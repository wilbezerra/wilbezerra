# 🔐 WSB Flash Clean — Privacy Policy (v3.1)

---

## 📌 Data Collection
This software **does not collect, transmit, or share any personal data**.

---

## 💻 Local Processing
All operations are performed **locally on the user's machine**.  
No external communication, cloud sync, or telemetry is used.

---

## 🔐 Authentication Data Protection
The cleaning engine is designed to **preserve critical authentication data**, including:

- Cookies
- Login Data
- Web Data
- Local State
- Sessions
- Persisted tokens
- Critical application preferences

---

## 🧠 Operational Safety
Recent revisions introduced:

- Safe icon cleaning mode that preserves desktop visual preferences
- Aggressive icon rebuild mode for deeper repair scenarios
- Explicit protections for application profiles, storages, extensions and sensitive data

---

## 🚀 Public Release Protection
The public GitHub release may include:

- File integrity validation (self-hash)
- Local violation logging
- Safe fallback blocking when tampering is detected

> Note: These protections are **only applied to public protected builds**, preserving full editability in development versions.

---

## 📝 Logs
Logs are stored locally under user-accessible folders used by the application.

- No telemetry is used
- No data is transmitted externally

---

## ✅ Summary

WSB Flash Clean:
- Operates 100% locally
- Preserves authentication and critical data
- Does not collect or share user information
- Uses integrity validation only in designated public builds
